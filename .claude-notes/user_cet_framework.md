User is Evan Ye, author of the "From Prediction to Self" paper (arXiv:2606.05605). This is Paper 1 in a broader series built around a theoretical framework the user calls **CET (Conditional Emergence Theory)**.

CET §13.8 provides a **condition-set expansion logic** for emergence: each new level of emergence requires two conditions simultaneously:
1. **Theoretical necessity**: `I(X; S_{t+1} | existing conditions) > 0` — the new variable X carries non-redundant information for predicting the target given what's already in the condition set.
2. **Architectural necessity**: an information pathway must exist that lets X reach the predictive computation.

The paper's 4-stage developmental sequence (persistent state → causal loop → proprioception → async awakening) is one instance of this condition-set expansion applied to self-emergence.

**How to apply**: When discussing experimental design with this user, filter proposals through the CET lens — is the new element (a) information-theoretically necessary given what's already in the condition set, and (b) implemented as an architectural pathway (not a supervision signal). Design proposals that "inject via loss" rather than "provide via input channel" will be rejected as violating CET's "conditional emergence, not forced injection" principle.
